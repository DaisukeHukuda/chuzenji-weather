# 中禅寺湖 天気アプリ 設計書

作成日: 2026-06-21

## 1. 目的・背景

SUPツアー運営のため、**中禅寺湖（湖面標高 1269m）のピンポイント予報**を社内で素早く確認できるWebアプリを作る。

多くの天気サービスは「日光」という地点で値を返すが、これは中禅寺湖より約1000m低い市街地（東照宮あたり）の値で、湖上のSUP判断には使えない。そこで**標高を補正した予報**を表示する。

- 利用範囲: **社内専用・非商用**（一般公開しない）
- 対象地点: **中禅寺湖の1地点固定**
- 主目的: 予報の確認（現在の実況表示は不要）

## 2. データソース

### Open-Meteo Forecast API

緯度経度に加え `elevation` を渡すと、その標高に合わせて気温・気圧などを自動補正する。JMA（気象庁）モデルの予報を中禅寺湖の標高で補正して取得する。非商用・社内利用のため**無料枠・認証不要**で利用する。

- エンドポイント: `https://api.open-meteo.com/v1/forecast`
- 主なクエリパラメータ:
  - `latitude=36.727` / `longitude=139.477`（中禅寺湖。最終値は実装時に確定）
  - `elevation=1269`（標高補正）
  - `models=jma_seamless`（JMAモデル）
  - `timezone=Asia/Tokyo`
  - `wind_speed_unit=ms`（**m/s** 表示）
  - `forecast_days=16`（最大16日）
  - `hourly=weather_code,temperature_2m,wind_speed_10m,wind_direction_10m,wind_gusts_10m,precipitation,precipitation_probability,cloud_cover,uv_index`
  - `daily=weather_code,temperature_2m_max,temperature_2m_min,wind_speed_10m_max,wind_gusts_10m_max,wind_direction_10m_dominant,precipitation_sum,precipitation_probability_max,uv_index_max,sunrise,sunset`

### 表示する気象要素（10項目）

天気・気温・風速・風向・最大風速（突風）・降水量・降水確率・雲量・UV指数・日の出/日の入り。
※ 体感温度・湿度・気圧は表示しない。

### 出典表示

ライセンス要件として「出典: Open-Meteo（JMAモデル・標高補正）」を画面に明記する。

## 3. 画面構成（Windfinder風マトリクス・モバイルファースト）

スマートフォンでの利用をメインのユースケースとする。縦画面・1カラムを基本とし、PCでも中央に程よい幅で破綻なく表示する。

### レイアウト

- **行＝気象要素／列＝時刻**のマトリクス表（Windfinder風）。
- **左端の項目ラベル列は固定**し、**時刻方向（横）にスクロール**して先の時間を見る。スマホでもWindfinderと同じ情報密度を確保する。
- **日ごとに見出し**（例: `6/21（日）`）で区切る。

### 行（上から）

時刻 / 天気（アイコン）/ 風向（**回転矢印**）/ 風速 / 最大風速 / 降水量 / 降水確率 / 雲量 / 気温 / UV指数。

### 視覚表現

- **風向**: 風が吹いていく方向を回転矢印で表現。
- **突風（最大風速）・気温**: 値に応じた**色分けセル**（Windfinder同様のグラデーション）。
- **天気・降水**: アイコン表示。

### 粒度タブ

画面上部のタブで時間粒度を切り替える。各タブの中身がそれぞれ上記マトリクスになる。

| タブ | 期間 | 列の単位 | データ元 |
|------|------|----------|----------|
| 1時間 | これから48時間 | 1時間ごと | hourly そのまま |
| 3時間 | これから5日 | 3時間ごと | hourly を3hに集計 |
| 半日 | 7日 | 午前/午後 | hourly を午前(0–12)/午後(12–24)に集計 |
| 1日 | 7日 | 1日ごと | daily |
| 週 | 16日俯瞰 | 1日ごと | daily（16日を1画面で俯瞰） |

集計（3時間・半日・週）は**追加APIを叩かず、取得済みデータからブラウザ側で計算**する。集計ルール（暫定・実装時に微調整）:
- 気温: バケット代表値（時間別は実気温、日別は最高/最低）
- 風速・突風: バケット内の最大値
- 風向: バケット内の代表（最頻 or 風速最大時）方向、日別は `wind_direction_10m_dominant`
- 降水量: バケット内の合計
- 降水確率: バケット内の最大
- 雲量・UV: バケット内の平均 or 代表値

## 4. 自動更新

山の天気は変わりやすいため、更新頻度を優先する。**ブラウザの手動リロードは不要**で、ページが自動更新する。

- **5分ごと**にAPIを再取得し、DOMの数値・色・アイコンだけを差し替える（全体再描画によるチラつきを避ける）。
- **タブにフォーカスが戻った時**（`visibilitychange`）・**通信が復帰した時**（`online`）にも即時更新する。
- **「最終更新 HH:MM」＋次回更新までのカウントダウン**をヘッダーに表示する。
- 待てない時用に**手動更新ボタン**も設置する。
- 補足: Open-Meteoの元データ（JMAモデル）自体の更新は数十分〜数時間ごと。5分ポーリングは「変化を取りこぼさない」ための間隔であり、毎回新しい値が出るとは限らない。

## 5. アーキテクチャ・デプロイ

- **静的1ページ構成**（HTML / CSS / JS）。サーバ・APIキー不要。ブラウザから直接Open-Meteoを呼ぶ。
- ビルドツールは最小限（必要なら軽量なバンドラ／不要ならなしでも可）。
- **Cloudflare Pages** にデプロイし、社内でURLを共有。各自のスマホ/PCで開く。ホーム画面に追加すればアプリのように使える。

### モジュール構成（責務分離）

| モジュール | 責務 | 依存 |
|------------|------|------|
| `api` | Open-Meteoへのfetch・パラメータ組み立て・生JSON取得 | fetch |
| `transform` | 生データ→各粒度（1h/3h/半日/1日/週）への集計、天気コード→アイコン、風向→16方位＋回転角、単位整形、色スケール算出 | （純粋関数・副作用なし） |
| `render` | マトリクス表のDOM生成・差し替え | transform |
| `refresh` | 5分タイマー・visibility/online連動・エラー時の挙動制御 | api, render |
| `app` | 初期化・タブ状態管理・各モジュールの結線 | 全て |

`transform` は外部依存のない純粋関数群とし、単体でテスト・理解できるようにする。

## 6. エラーハンドリング

- API失敗・タイムアウト時はリトライ。失敗が続く場合は**直近の正常データを保持**して表示し続け、「最終更新が古い」状態をヘッダーに明示する。
- 一部項目が欠損している時刻は「—」表示にし、表全体は壊さない。
- 通信オフライン時はバナー等で状態を示し、復帰時に自動再取得する。

## 7. テスト方針

- `transform` の純粋関数を中心に**ユニットテスト**（vitest）:
  - hourly → 3時間 / 半日 / 週 の集計ロジック
  - 天気コード（WMO）→ アイコンのマッピング
  - 風向（度）→ 16方位ラベル＋矢印回転角
  - 突風・気温の値 → 色スケール
  - 単位・桁の整形
- `api` のパラメータ組み立てを検証し、fetchはモックする。
- 代表的なレンダリング（マトリクス生成、差し替え）も対象に含める。

## 8. スコープ外（YAGNI）

- 現在の実況表示（アメダス奥日光など）
- 複数地点の切り替え・任意地点指定
- 実況や気圧・体感温度・湿度の表示
- 一般公開・商用配信・認証/ログイン
- 通知・アラート機能
